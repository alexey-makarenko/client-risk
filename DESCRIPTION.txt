Есть сущность Client {id: , riskProfile }
riskProfile может принимать значения LOW, NORMAL, HIGH
Нужно разработать сервис по учету таких клиентов, чтобы с ним можно было взаимодействовать по HTTP/REST.

1) Нужен обычный CRUD-сервис. Т.е. добавить профиль, обновить, получить, удалить.
2) Также нужен метод «слияния», который объединяет клиентов, присваивая им максимальный (среди этих клиентов) уровень риска.

Хранилище - InMemory.
